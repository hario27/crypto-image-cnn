# crypto-image-cnn

**市場のミクロ構造を画像として描画し CNN で学習することで、短期の暗号資産リターンを予測する — 研究用スキャフォールド。**

このリポジトリの狙いは、*クリーンでリークのない* 研究の土台を提供することであって、
儲かる戦略を提供することではない。平坦な特徴量ベクトルをモデルに渡すのではなく、
オーダーフロー特徴量の30分窓を **2D画像** のスタック(Gramian加算フィールド、
ウェーブレットのスカログラム、1秒毎の約定回数の生マップ)に変換し、その上で CNN を
学習する点が中心テーマである。

> ⚠️ **投資助言ではなく、儲かる戦略でもない。** BTC無期限の10秒リターンの予測は極めて
> 難しく、誠実な test R² は小さい。

## 設計方針 — 意図的に最小限

精度を狙った特徴量(テクニカル指標、取引所間のリードラグ、funding などのエッジ要素)は
**あえて入れていない**。データは Binance 単独、特徴量は生に近い最小限のみ。

**エッジの探索 — 特徴量設計、データ追加、ラベル改善 — はこのリポジトリを見る人に委ねる。**
ここが提供するのは「正しく(リークなく)実験するための土台」であり、その上に何を載せるかは
読者次第である。

## 着想

```
生の約定 ─▶ N秒足 ────▶ 最小特徴量 ──▶ 30分窓 ────▶ 画像 ─────▶ CNN ─▶ 次バーのリターン
(Binance)  (OHLCV +     (生に近い・        (窓 =        (6ch ×
            フロー+1s)    先読みなし)         画像の一辺)   180×180)
```

* **生の約定から構築。** バーは Binance の公開約定アーカイブから集計するので、kline
  エンドポイントが捨てるミクロ構造 — taker の買い/売り内訳と、バー内の *1秒毎* の
  約定回数ヒストグラム — を保持する。
* **最小の特徴量。** log_return、バー形状(高低比・実体比)、フロー不均衡、出来高変化、
  1秒毎カウントの変化 — いずれもバー t と t-1 だけから作る点演算 / 1階差分で、
  構造的にリークしえない。
* **画像チャンネルは情報ではなく表現を足す。** 窓を画像化しても、窓が既に含む情報を
  再形成するだけ。つまりこのリポジトリが問うのは: *この種のデータに対して CNN の
  帰納バイアスは効くのか?* ということ。

## 数値を信頼できる理由(リーク対策)

金融MLはリークで生き死にが決まる。このリポジトリはリークを起こしにくいように作ってある:

| 対策 | 場所 |
|---|---|
| **特徴量は点演算 / 1階差分のみ** — 構造的に未来を参照しない。未来改変テストで「未来を変えても過去の特徴量が変わらない」ことを表明。窓特徴量を足す場合の因果性の注意も記載。 | `src/features.py`, notebook §3 |
| **ターゲットは *過去側* ボラで正規化** — 未来を覗くのは分子(ラベル)だけ。 | `src/target.py` |
| **スケールより前に分割、scaler は train のみで fit、窓は分割境界をまたがない。** | `src/pipeline.py` |
| **test は最後に1回だけ** 触る(val でモデル選択した後) — test でチューニングしない。 | `src/train.py` |
| **画像は窓だけから生成** — シーケンスと同じ先読みなしの保証。 | `src/images.py` |

## 構成

```
notebook/tutorial.ipynb   全体の物語: 流れ + 可視化(これを実行する)
src/
  config.py               全パラメータを1つのdataclassに
  data.py                 Binance公開APIの約定取得 -> N秒足(キャッシュ付き)
  features.py             最小・因果的な特徴量(エッジ探索は読者に委ねる)
  target.py               ボラティリティ正規化した forward return
  pipeline.py             バー -> 特徴量 -> 分割 -> スケール -> シーケンス(キャッシュ)
  images.py               時系列 -> マルチチャンネル画像(オンザフライ) + Dataset
  models.py               SmallCNN / ResNet18 回帰モデル
  train.py                valで選択する学習 + test1回の評価
smoke_test.py             合成データでのCPU専用 一気通貫テスト
```

## クイックスタート

```bash
# 1. 依存(TA-Lib などのシステム依存は不要)
pip install -r requirements.txt          # torch は pytorch.org から CUDA に合うものを入れる
# 2. (任意)ネットワーク・GPUなしでコード全体を検証
python smoke_test.py
# 3. チュートリアルを実行
jupyter lab notebook/tutorial.ipynb
```

`src/config.py` の既定値は `BTCUSDT` の10秒足 **3ヶ月ぶん** を取得する — そこそこのGPU
1枚で学習できる量。データとモデルは `data/` と `models/`(gitignore済み)にキャッシュ
され、**notebook を再実行しても既存のものは再取得・再計算しない**。期間を変えたい場合は
`src/config.py` の `start` / `end` を編集する。

## 結果と、その先

再現は notebook を実行する。test R² は **小さく**、方向一致率は 0.5 付近を想定すること —
最小特徴量のニュートラルなベースラインなので当然である。ここからエッジを足すのが読者の
仕事になる。例えば:

* **特徴量** — テクニカル指標、レジーム変数、自分の発想(窓を使うものは因果性に注意)。
* **データ** — 板の深さ/不均衡(本来的に2Dで、ネイティブな画像)、他取引所、funding、清算。
* **ラベル** — mid-to-mid の次バーリターンではなく、約定を考慮したラベル。
* **評価** — コストを考慮したバックテスト(maker / taker / 約定保証)と四半期毎の頑健性。
